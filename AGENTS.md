# Repository Guidelines

## プロジェクト構成とモジュール配置

このリポジトリは、アイドルマスター公式イベントスケジュールを Google スプレッドシートへ取り込み、Google カレンダーへ同期する Google Apps Script プロジェクトです。現在の主な構成は次のとおりです。

- `README.md`: 目的、機能、対象ブランドの概要。
- `src/`: Apps Script の実装を配置するディレクトリ。例: `Code.gs`、スクレイピング処理、シート更新処理、カレンダー同期処理、共通ユーティリティ。
- `LICENSE`: ライセンス。

本番用コードは `src/` 配下に置いてください。テストを追加する場合は `test/` または `tests/` を作成し、`src/scheduleParser.gs` に対して `tests/scheduleParser.test.js` のように対応関係が分かる名前にします。

## ビルド・テスト・開発コマンド

現時点では `package.json` やビルド設定は未追加です。導入するまでは、Apps Script への反映手順を `README.md` に記録してください。

Node.js ベースの開発環境を追加する場合の推奨コマンド:

- `npm install`: 開発依存関係をインストールします。
- `npm test`: テストを実行します。
- `npm run lint`: フォーマットと静的解析を確認します。
- `npx clasp push`: ローカルの Apps Script ファイルを Google Apps Script プロジェクトへ反映します。
- `npx clasp pull`: リモート側の Apps Script 変更を取得します。

共有可能な状態になった場合のみ、`package.json`、ロックファイル、`.clasp.json` をコミットしてください。認証情報や個人用 ID はコミットしないでください。

## コーディング規約と命名規則

Google Apps Script 互換の JavaScript を使用します。インデントは 2 スペース、セミコロンあり、意味が明確な名前を基本とします。関数と変数は `camelCase`、クラスやコンストラクタ相当は `PascalCase`、定数は `UPPER_SNAKE_CASE` を使用してください。

スクレイピング、シート書き込み、カレンダー同期、ブランド判定は責務ごとに分けます。カレンダー ID、スプレッドシート ID、シークレット類をソースへ直書きしないでください。

## テスト方針

現時点でテストフレームワークは未導入です。パーサー、ブランド分類、日付・時刻処理、キャンセル検知、カレンダー予定更新を変更する場合は、あわせてテストを追加してください。

外部サイトへのライブアクセスに依存せず、保存済み HTML や JSON フィクスチャを使った再現性のあるテストを優先します。

## コミットとプルリクエスト

履歴は初期コミットのみのため、詳細な規約はまだありません。コミットメッセージは `Add schedule parser` や `Fix all-day event sync` のように、短い命令形で書いてください。

プルリクエストには概要、テスト結果、セットアップやデプロイ手順の変更を記載します。スプレッドシートやカレンダーの挙動が変わる場合は、スクリーンショットまたは変更前後の説明を含めてください。

## エージェント向け指示

このリポジトリでの応答は常に日本語で行ってください。コマンド、構成、開発手順が変わった場合は、このファイルも更新してください。
